# 전역 밸런스 시스템 소비 범위 결정

DUVU의 전역 밸런스 시스템은 기존 상위 원칙(intent, meaning, page pattern, layout restraint, component level)을 덮어쓰지 않고, 그것을 실제 화면 비례와 정렬로 번역하는 중간 계층으로 유지한다.

## 결정
- 전역 밸런스 시스템은 다음 축을 실제 렌더에 직접 연결한다.
  - spacing / rhythm
  - content width
  - title width
  - preview min width
  - alignment mode
  - code tab distribution
  - palette bias
- `journey`와 `density`는 계산 결과에 간접적으로만 녹이는 데서 멈추지 않고, 일부 반복 UI에서 직접 소비한다.
  - `density` → hero stats/template grid gap
  - `journey` → dense preview min width profile
- 반대로, 모든 UI 요소를 dataset 분기 기반으로 재작성하지는 않는다. 이미 계산된 CSS 변수로 충분히 설명되는 부분은 그 경로를 유지한다.

## 이유
- 이전 상태에서는 balance contract에 있는 일부 축이 선언만 되고 DOM에서 직접 소비되지 않아, 시스템 설명력은 높았지만 실행 밀도가 낮았다.
- 그렇다고 모든 컴포넌트에 별도 엔진을 붙이면 과도한 엔지니어링이 된다.
- 따라서 반복해서 문제가 재발한 축만 직접 소비하도록 연결하는 것이 유지보수와 품질의 균형점이다.

## 구현 결과
- `alignmentMode`는 hero/section/tab 정렬에 직접 연결됐다.
- `contentMaxWidth`는 article/quote/preview-single에 연결됐다.
- `previewMinWidth`는 dense preview grid에 연결됐다.
- `density` dataset은 hero stats/template grid gap에 연결됐다.
- `journey` dataset은 dense preview grid의 min width bias에 연결됐다.

## 주의사항
- 새 축을 추가할 때는 먼저 실제 재발 문제인지 확인한다.
- 단순히 "언젠가 쓸 수 있음" 수준의 축은 추가하지 않는다.
- palette 같은 특수 UI는 전역 밸런스 위의 하위 엔진으로 유지한다.
