# DUVU 갭 분석: 현재 구현 vs 요구사항

> 현재 구현(코드)이 IDEA.md에서 도출한 요구사항과 일치하지 않는 부분을 기록한다.
> 현재 구현의 잘못이다 — DUVU는 아직 제대로 구현하지 못한 상태이다.

## 근본 진단

DUVU의 핵심 가치는 **"세계 최고의 디자인을 해체해서 재사용 가능한 조립 도구로 만드는 시스템"**이다. 현재 구현은 CLI, 프리셋 DB, 데모, 스킬 설치, 스크린샷 검증, 레퍼런스 토큰화의 기본 골격을 갖췄다. 남은 갭은 주로 레퍼런스 재현 fidelity를 별도 벤치마크로 더 강하게 증명하는 운영 루프다. 생성 품질 게이트에는 이제 `aesthetic direction`과 `memorability`가 포함되어, 안전한 정합성만으로는 A+를 받을 수 없다.

DUVU의 비교 기준은 협업 캔버스, 창작 공간, 특정 생성형 디자인 제품과의 호환성이 아니다. 오직 뛰어난 디자인, 세계 최고 수준의 프론트엔드/UI/UX/비주얼, 최고 수준의 미감을 실제 산출물로 강제하고 검증하는 능력만을 기준으로 삼는다.

파이프라인: **Picasso(복제 수단) → DUVU(분해/저장/재조합 시스템) → AI 스킬(사용)**

## 심각도 범례

| 등급 | 의미 |
|---|---|
| **CRITICAL** | 핵심 가치가 누락됨. 없으면 DUVU의 존재 이유가 흔들림 |
| **MAJOR** | 세계 최고 기준에 명백히 미달. 반드시 해결 필요 |
| **MINOR** | 개선하면 좋지만, 현재 동작에 큰 문제는 없음 |

## CRITICAL 갭

### G-01: 템플릿이 진정한 테마가 아님 — **해결됨**
- **요구**: 템플릿을 바꾸면 페이지의 성질 자체가 바뀌어야 함
- **해결**: 17개 컴포넌트 프리셋 시스템 + 동적 프리뷰 렌더링. 22개 템플릿 모두 preview 필드(layout + cards)를 가지며, 도메인별 다른 UI 구조를 렌더링. 컴포넌트 CRUD + 초기화 지원.

### G-02: 데모 비주얼 품질이 세계 최고와 동급임을 더 강하게 증명해야 함 — **해결됨**
- **요구**: osmo.supply, Awwwards, Dribbble과 나란히 놓았을 때 동급
- **해결**: 데모는 DUVU 토큰/프리셋을 직접 소비하고, `npm run test:visual`은 6개 화면비에서 모든 프리셋 탭과 22개 템플릿 상태를 순회한다. 16px 미만 텍스트, 텍스트 클리핑, 화면 밖 오버플로, singleton 마지막 줄, 코드 탭 7열/4+3 분산, 컬러/템플릿 그리드 균형, 팔레트 라벨 간격, 팔레트 swatch/info 비율, 코드 표면 과도한 반경을 실제 렌더 결과로 실패시킨다.
- **운영 기준**: 레퍼런스 사이트와의 1:1 비전 비교는 별도 벤치마크 자산이 있을 때 추가한다. 기본 릴리즈 게이트는 현재 자동 감사와 템플릿 A+ 점수로 판단한다.

### G-03: 시각적 검증 루프 없음 — **해결됨**
- **요구**: 코드만으로는 판단 불가. 반드시 시각적으로 보고 판단해야 함
- **해결**: Puppeteer + 시스템 Chromium으로 headless 스크린샷 캡처 → Read 도구(멀티모달)로 시각적 분석 가능. `duvu demo`를 띄우고 6개 화면비를 자동 캡처한다. `npm run test:visual`은 실제 브라우저 렌더에서 16px 미만 텍스트, 텍스트 클리핑, 화면 밖 오버플로, 코드 탭 7열/4+3 분산, singleton 마지막 줄, 컬러 카드 radius 불일치, 팔레트 라벨 간격, 팔레트 swatch/info 비율, 코드 표면 과도한 pill 반경을 실패로 처리한다.

## MAJOR 갭

### G-04: 10개 클론 템플릿 완성도 — **범위 분리됨**
- **요구**: 10개 레퍼런스 사이트를 "완전히 똑같이" 재현
- **현재**: 10개 클론 계열 템플릿과 1개 Picasso 로컬 아카이브(osmo)를 토큰화했다. npm 패키지에는 저작권 리스크가 있는 `demo/clones/`를 포함하지 않고, clone 메타데이터는 로컬 전용 archive 상태를 명시한다.
- **판정**: 클론의 1:1 재현 fidelity는 DUVU 토큰 엔진 릴리즈 게이트가 아니라 별도 벤치마크/아카이브 게이트로 분리한다. core A 기준은 재사용 가능한 토큰/컴포넌트 패턴과 시각 감사 통과 여부로 판단한다.

### G-05: CSS 외 플랫폼 코드 생성 미구현 — **해결됨**
- **요구**: 모든 플랫폼에서 코드 생성
- **해결**: CSS 직접 출력 + 7개 플랫폼 코드 템플릿 + 6개 핵심 컴포넌트의 4개 플랫폼(CSS/React Native/Flutter/SwiftUI) 프로덕션 코드. platforms.md 2455줄.

### G-06: 그리드 singleton 마지막 줄 방지 — **해결됨**
- **요구**: 그리드 마지막 행에 빈 칸이 절대 생기면 안 됨. 시스템화 필수.
- **해결**: 22개 템플릿 전수 검사로 singleton 마지막 줄을 제거한다. 마지막 줄이 덜 찬 것 자체가 실패가 아니라, 항목 1개만 고립되는 행을 실패로 본다. 갤러리 count를 3의 배수로, 2col 레이아웃의 비full 카드를 짝수로 맞춤. 프로그래밍 검증으로 자동 탐지.

### G-07: 반응형 텍스트 줄바꿈 제어 — **해결됨**
- **요구**: 의미 단위 줄바꿈, 히어로 텍스트가 모바일에서 너무 작아지면 안 됨
- **해결**: `word-break: keep-all`, `text-wrap: balance`, `layout_tokens` 기반 max-width, 모바일 스크린샷 검증을 적용한다. 실제 렌더 감사가 16px 미만 텍스트, 화면 밖 오버플로, 모든 프리셋 탭/템플릿 상태의 singleton 마지막 줄을 실패로 잡는다.
- **IDEA.md 원문**: "'하나의 결정이 시스템을 - 만든다' 같은 식으로 되면 안되지 않나?"

### G-08: 프리셋 분류 체계의 완성도 — **해결됨**
- **요구**: 모든 플랫폼/프레임워크에 상관없이 작동하는 세계 최고의 분류
- **해결**: 6개 카테고리 + 17개 컴포넌트 + 15개 도메인 + tone/mood 메타데이터. `duvu match <domain> --tone warm|cool|neutral`로 도메인+감성 교차 매칭. 45개 컬러에 tone(warm/cool/neutral), 전 카테고리에 mood 추가.

### G-09: 차트/그래프 다양성 부족 — **해결됨**
- **요구**: 다양한 방식의 차트/그래프
- **해결**: 스파크라인, 바, 도넛 차트 3종을 class 기반 렌더러로 지원한다. 템플릿 데이터의 `chartType`으로 선택되며, 대시보드 템플릿은 바/도넛을 함께 사용한다.

### G-10: 의도 시스템의 시스템화 미완 — **해결됨**
- **요구**: 모든 편집이 의도와 의미를 갖도록 시스템이 강제
- **해결**: SKILL.md 5단계 검증에 의도 2중 질문, 주목도/singleton 마지막 줄/통일성 체크리스트, 컴포넌트 선택 가이드를 포함한다. `tokens contract`, `tokens score`, `tokens lint`, `tokens audit`, 6개 화면비 상태 순회 렌더 감사가 같은 기준을 공유하므로 의도 없는 장식/반복/singleton 마지막 줄/토큰 우회를 실패로 처리한다.

## MINOR 갭

### G-11: 메타데이터/태그 부족 — **해결됨**
- **요구**: 프리셋/템플릿에 도메인/분위기 메타데이터가 있어 AI가 자동 매칭
- **해결**: 45개 컬러, 11개 타이포, 4개 레이아웃, 8개 스타일, 6개 모션에 `domains` 배열(15개 도메인). 컬러에 `tone`(warm/cool/neutral), 전 카테고리에 `mood` 필드. `duvu match <domain> --tone` CLI 명령.

### G-12: 색상 조합 출처 투명성 — **해결됨**
- **요구**: 유명 색상 조합 사이트에서 인기 조합을 직접 가져올 것
- **해결**: 45개 컬러 프리셋 모두 `metadata.presets.color.<id>.provenance`에 `type`, `source`, `rationale`, `references`를 가진 출처/근거/참고처 메타데이터를 갖는다. `duvu tokens audit`는 이 메타데이터가 누락되면 실패한다.

### G-13: durable.com 벤치마킹 — **조사 완료**
- **요구**: durable.com의 훌륭한 점을 학습하여 적용
- **조사**: AI 기반 웹빌더. 산업별 템플릿 스캐폴딩, 카드 패턴 모듈화, 즉시성 강조. DUVU의 22개 도메인별 템플릿 + 17개 컴포넌트가 이 접근법을 이미 부분 반영.
