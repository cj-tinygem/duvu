# DUVU 갭 분석: 현재 구현 vs 요구사항

> 현재 구현(코드)이 IDEA.md에서 도출한 요구사항과 일치하지 않는 부분을 기록한다.
> 현재 구현의 잘못이다 — DUVU는 아직 제대로 구현하지 못한 상태이다.

## 근본 진단

DUVU의 핵심 가치는 **"세계 최고의 디자인을 해체해서 재사용 가능한 조립 도구로 만드는 시스템"**이다. 현재 구현은 이 시스템의 **껍데기**(CLI, 프리셋 저장/조회, 데모 페이지)만 있고, **본질**(레퍼런스 → 분해 → 저장 → 재조합)이 없다. 아래 13개 갭은 전부 이 본질의 부재에서 파생된 것이다.

파이프라인: **Picasso(복제 수단) → DUVU(분해/저장/재조합 시스템) → AI 스킬(사용)**

---

## 심각도 범례

| 등급 | 의미 |
|---|---|
| **CRITICAL** | 핵심 가치가 누락됨. 없으면 DUVU의 존재 이유가 흔들림 |
| **MAJOR** | 세계 최고 기준에 명백히 미달. 반드시 해결 필요 |
| **MINOR** | 개선하면 좋지만, 현재 동작에 큰 문제는 없음 |

## CRITICAL 갭

### G-01: 템플릿이 진정한 테마가 아님 — **해결됨**
- **요구**: 템플릿을 바꾸면 페이지의 성질 자체가 바뀌어야 함
- **해결**: 14개 컴포넌트 프리셋 시스템 + 동적 프리뷰 렌더링. 22개 템플릿 모두 preview 필드(layout + cards)를 가지며, 도메인별 완전히 다른 UI 구조를 렌더링. 컴포넌트 CRUD + 초기화 지원.

### G-02: 데모 비주얼 품질이 세계 최고에 미달
- **요구**: osmo.supply, Awwwards, Dribbble과 나란히 놓았을 때 동급
- **현재**: 기본적인 프리셋 탐색기 수준. 의도성, 과감성, 독창성 부족.
- **IDEA.md 원문**: 반복적으로 "이게 세계 최고야?", "냉정하게 생각해봐" 라는 피드백

### G-03: 시각적 검증 루프 없음 — **해결됨**
- **요구**: 코드만으로는 판단 불가. 반드시 시각적으로 보고 판단해야 함
- **해결**: Puppeteer + 시스템 Chromium으로 headless 스크린샷 캡처 → Read 도구(멀티모달)로 시각적 분석 가능. `duvu demo`를 띄우고 데스크톱/모바일 뷰를 자동 캡처하여 검증.

## MAJOR 갭

### G-04: 10개 클론 템플릿 완성도 낮음
- **요구**: 10개 레퍼런스 사이트를 "완전히 똑같이" 재현
- **현재**: 색상/분위기만 가져온 수준. 레이아웃/구조 클론이 아님.
- **누락 사이트**: ssyenc.com, tscent-tea.com — 현재 구현에 아예 미포함

### G-05: CSS 외 플랫폼 코드 생성 미구현
- **요구**: 6개 플랫폼 모두에서 코드 직접 생성
- **현재**: CSS만 직접 출력. 나머지 5개는 "참조하세요" 메시지.
- **참고**: AI 스킬이 references/platforms.md를 참조하여 변환하는 간접 방식은 존재

### G-06: 그리드 orphan 방지 미완성
- **요구**: 그리드 마지막 행에 빈 칸이 절대 생기면 안 됨. 시스템화 필수.
- **현재**: 데모의 컬러 카드에만 일부 구현. 범용 시스템화 안 됨.
- **IDEA.md 원문**: "5-Layer 아키텍처 아래 카드는 5장인데 한칸이 비게 된다. 이런 상황을 만들지 말라고 했을텐데"

### G-07: 반응형 텍스트 줄바꿈 제어 미흡
- **요구**: 의미 단위 줄바꿈, 히어로 텍스트가 모바일에서 너무 작아지면 안 됨
- **현재**: CSS `word-break: keep-all`과 `clamp()`만 사용. 의미 단위 줄바꿈 제어 없음.
- **IDEA.md 원문**: "'하나의 결정이 시스템을 - 만든다' 같은 식으로 되면 안되지 않나?"

### G-08: 프리셋 분류 체계의 완성도
- **요구**: 모든 플랫폼/프레임워크에 상관없이 작동하는 세계 최고의 분류
- **현재**: 컬러/타이포/레이아웃/스타일/모션/그라디언트 — 6개 분류. 기본 구조는 있으나, 스타일 프리셋이 제공하는 옵션이 제한적.
- **추가 필요**: 정렬 방식, 아웃라인 스타일, 버튼 스타일 등 세분화된 프리셋

### G-09: 차트/그래프 다양성 부족 — **부분 해결**
- **요구**: 다양한 방식의 차트/그래프
- **현재**: 스파크라인 + 바 차트 2종. class 기반 다중 차트 시스템 구축. 도넛 등 추가 가능.

### G-10: 의도 시스템의 시스템화 미완
- **요구**: 모든 편집이 의도와 의미를 갖도록 시스템이 강제
- **현재**: SKILL.md와 rules.md에 원칙으로 기술되어 있으나, 자동 강제 메커니즘은 없음. AI의 "자율적 판단"에 의존.

## MINOR 갭

### G-11: 메타데이터/태그 부족
- **요구**: 프리셋/템플릿에 도메인/분위기 메타데이터가 있어 AI가 자동 매칭
- **현재**: 템플릿에 description, tags, industry, mood, complexity 필드 존재. 컬러 프리셋에는 cat 필드만.

### G-12: 색상 조합 출처 투명성
- **요구**: 유명 색상 조합 사이트에서 인기 조합을 직접 가져올 것
- **현재**: 41개 컬러 프리셋은 브랜드(Toss, Stripe 등) + 자연색 기반. 이론적/과학적 근거가 명시되지 않은 것도 있음.

### G-13: durable.com 벤치마킹 미완
- **요구**: durable.com의 훌륭한 점을 학습하여 적용
- **현재**: 미착수
