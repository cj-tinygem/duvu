# DUVU PRD (Product Requirements Document)

> 이 문서는 IDEA.md(1990줄)의 전체 피드백에서 중복을 제거하고 요구사항을 도출한 정본이다.
> IDEA.md 원문은 보존하되, 이 문서가 현재 요구사항의 근거다.

---

## 1. 제품 정의

**DUVU**는 세계 최고 수준의 UI/프론트엔드를 자동으로 생산하는 **범용 디자인 시스템 엔진**이다.

- 사람이 CLI로 직접 사용할 수 있다
- AI 에이전트(Claude Code, Codex CLI, Gemini CLI)가 스킬로 자동 발동하여 사용한다
- "duvu를 사용해서 주식매매 프로그램을 개발해줘" 같은 요청에 AI가 바로 최적의 디자인을 적용할 수 있어야 한다
- MCP 방식은 사용하지 않는다. 스킬 기반으로만 동작한다

### 핵심 가치

DUVU는 **세계 최고의 디자인을 해체해서 재사용 가능한 조립 도구로 만들어주는 DB/라이브러리**다.

---

## 2. 최종 검증 기준 (비타협)

아래 3가지를 **동시에** 만족해야 최소 통과 기준이다.

### 기준 1: CRAP 4원칙
DUVU로 만든 모든 UI는 CRAP(Contrast, Repetition, Alignment, Proximity) 4원칙이 지켜지는지 실전 테스트로 검증해야 한다.

### 기준 2: 레퍼런스 동급
아래 레퍼런스 사이트들과 비교했을 때 **절대 뒤쳐지지 않는** 퀄리티여야 한다.

| 카테고리 | 사이트 |
|---|---|
| Gold Standard | osmo.supply |
| 종합 최고 | godly.website, bestawards.co.nz |
| 갤러리 | framer.com/gallery, gdweb.co.kr |
| 프로덕트 UI/UX | Refero.design, Mobbin, Pageflows |
| 비주얼/컨셉 | Dribbble, Behance |
| 웹/랜딩 | Awwwards, SiteInspire, Lapa Ninja |
| 컴포넌트/패턴 | UI Garage, Design Vault, Collect UI, Figma Design System Gallery |
| 웹 빌더 참고 | durable.com |

### 기준 3: 3중 규격 준수
Apple HIG + Material Design 3 + WCAG 2.1 AA를 **모든 항목에서 완벽히 동시 충족**해야 한다.

## 3. 디자인 철학 (요구사항으로서)

> 상세 철학은 [DESIGN-PHILOSOPHY.md](DESIGN-PHILOSOPHY.md) 참조.

DUVU가 생산하는 모든 디자인은 다음을 **시스템적으로 강제**해야 한다:

1. **의도 시스템** — 모든 요소(padding 1px, 색상 1%, opacity 0.01)에 방어 가능한 의도가 있어야 한다. 의도 없는 편집은 금지. "이건 왜 좋은가? 정말 좋은가?"를 모든 요소에 묻는다.
2. **비움의 미학** — 모든 요소에 "이게 없으면 안 되나?"를 묻는다. 쓸데없는 복잡성이나 과한 버무림이 없다.
3. **과감한 3요소** — 과감한 배치, 과감한 강조, 과감한 생략.
4. **절제** — 그라디언트, 애니메이션, 아웃라인, 그림자 남발 금지. 필요할 때만.
5. **ZERO 아웃라인** — 테두리/아웃라인은 절대적으로 필요한 경우에만. 영역 구분은 색차/음영으로.
6. **레벨과 레이어** — 같은 레벨의 요소는 같은 주목도. 주목도 계층("주목도")을 시스템화.
7. **상호작용 고려** — 모든 요소의 상호작용을 고려한다. 하나를 바꿨을 때 다른 것에 미치는 영향까지.
8. **아름다움 우선** — "아름다움은 첫 번째 관문이다. 이 세상에 못 생긴 수학이 설 자리는 없다."

## 4. 시스템 아키텍처 요구사항

### 4.1 디자인 토큰

- HIG/MD3/WCAG를 시스템적으로 준수하도록 설계
- `--duvu-` 접두사 토큰 체계
- 색상, 타이포, 간격, 모양, 엘리베이션, 모션, 강조계층, 그라디언트, 시맨틱, 인터랙션 — 10개 카테고리

### 4.2 프리셋 (개별 선택 + 조합 가능)

각 카테고리의 프리셋이 **독립적으로 존재**하고 **자유롭게 조합** 가능해야 한다.

| 카테고리 | 현재 수량 | 요구사항 |
|---|---|---|
| 컬러 | 41개 | 30개 이상의 검증된 색상 조합. 이론적/경험적으로 조화로운 것. 유명 색상 조합 사이트에서 인기 조합 수집. |
| 타이포 | 10개 | 각 폰트에 공감각적 의미 부여. 한국어 기본은 Pretendard. |
| 레이아웃 | 4개 | Compact, Comfortable, Spacious, Dense |
| 스타일 | 8개 | 라운드, 섀도우, 버튼 모양 등 비주얼 프리셋 |
| 모션 | 5개 | 목적 있는 모션만. 장식용 금지. |
| 그라디언트 | 9개 | 절제된 세련된 그라디언트. 단색 배경이 기본, 그라디언트는 옵션. |

**프리셋 품질 요구사항:**
- 모든 컬러 프리셋 전수조사: 눈 아픈 색 없음, 조화롭지 않은 조합 없음
- accent 위 텍스트 색상(흰/검) 자동 결정이 모든 프리셋에서 정확해야 함
- 다크/라이트 모드 양쪽에서 검증
- 유명 색상 조합 사이트에서 인기 있는 조합 수집하여 추가

### 4.3 템플릿 (종합 조합)

> **핵심 구분**: 템플릿 ≠ 컬러 프리셋. 템플릿은 **모든 프리셋의 종합 조합이자 완성된 하나의 테마**다.

- 템플릿을 바꾸면 **페이지의 성질 자체가** 바뀌어야 한다 (단순 색상 변경이 아님)
- 컬러 + 타이포 + 레이아웃 + 스타일 + 모션 + 그라디언트가 한 번에 세팅된 **ready-made 비주얼**
- 도메인/분위기에 맞는 메타데이터/태그 필요 (AI가 자동 매칭할 수 있도록)
- 30개 이상의 다양한 스타일: SaaS, 금융, 커머스, 포트폴리오, 럭셔리, 미니멀, 아날로그, 책 스타일, 예술적 스타일 등

### 4.4 10개 클론 레퍼런스 템플릿

아래 사이트를 **최대한 똑같이** 재현하여 템플릿으로 제공해야 한다. 색상만 가져오는 것이 아니라 레이아웃/스타일/모션을 포함한 완전한 클론.

1. museumofmoney.com
2. adambricker.com
3. oci.madebybuzzworthy.com
4. on.energy
5. griflan.com
6. haiqu.ai
7. rom.on.ca
8. ssyenc.com/kr/index (추가 근거 필요 — 현재 구현에 미포함)
9. tscent-tea.com (추가 근거 필요 — 현재 구현에 미포함)
10. osmo.supply

### 4.5 확장성

- 프리셋/토큰/템플릿을 사용자 또는 AI가 추가/삭제/관리 가능
- 기본값은 항상 복원 가능 (`duvu reset`)
- DB/라이브러리로서 저장, 검색, 열람 기능

## 6. 반응형 요구사항

- 모든 화면비(가로 넓은 → 가로 좁은 모바일)에서 완벽 대응
- **텍스트 줄바꿈**: 의미 단위로 균형 잡힌 줄바꿈 (word-break: keep-all). "하나의 결정이 시스템을 - 만든다" (X) → "하나의 결정이 - 시스템을 만든다" (O)
- **히어로 텍스트**: 모바일에서 너무 작아지면 안 됨. 알고리즘 보정 필요.
- **그리드 orphan 방지**: 마지막 행에 빈 칸이 생기면 안 됨. 열 수를 자동 조정하여 시스템적으로 방지.
- **터치 타겟**: 모바일에서 클릭 요소/글자가 너무 작으면 안 됨 (44px 최소)
- 참고 레퍼런스: osmo.supply의 반응형 대응 전략

## 7. 시각적 검증 요구사항

- 코드만으로는 디자인 품질을 판단할 수 없다
- **반드시 시각적으로(비전/멀티모달) 확인**하고 판단해야 한다
- 데모 웹페이지(`duvu demo`)가 시각적 검증의 주요 수단
- 스스로 9점 이상이 아니면 보여주지 말 것 (자기 평가에 관대하지 말 것)
- 모든 프리셋/템플릿을 하나하나 눈으로 직접 확인하는 전수조사 필요

## 8. CLI 및 도구화 요구사항

### 8.1 CLI 명령어

```
duvu list / show / info          # 조회
duvu generate / template          # 코드 생성
duvu add / remove / reset         # 확장/관리
duvu install-skill               # AI 스킬 설치
duvu demo                        # 데모 웹페이지
```

### 8.2 AI 스킬

- Claude Code: `~/.claude/skills/duvu/SKILL.md`
- Codex + Gemini: `~/.agents/skills/duvu/SKILL.md`
- 스킬이 발동되면 모든 UI 코드가 DUVU 토큰 체계를 따라야 함
- AI가 요구사항을 분석 → 프리셋 자동 매칭 → 코드 생성 → 검증의 파이프라인

### 8.3 데모 웹페이지

- 현재 프리셋/템플릿을 한눈에 확인하는 인터랙티브 쇼케이스
- 프리셋 선택 시 **현재 페이지 전체**가 해당 설정으로 바뀌는 방식이 직관적 (v18.6 스타일)
- 모든 프리셋 버튼이 동작해야 함 (일부만 클릭 가능한 상태 금지)
- 라이브 코드 출력 (CSS, Tailwind, Flutter, SwiftUI, Compose)

## 9. 성능 요구사항

- 웹 디자인/프론트엔드/UI의 성능을 항상 고려
- 불필요한 모션, 그라디언트, 그림자 등 성능에 영향을 주는 요소 최소화
- 외부 의존성 0개 유지 (순수 Node.js)

---

## 10. 차트/그래프 요구사항

- 다양한 방식의 차트/그래프를 그릴 수 있어야 함
- 현재는 하나의 스파크라인만 존재 (부족)
